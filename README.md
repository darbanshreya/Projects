## ML Lifecycle

This project follows the complete ML lifecycle:


### 1. Data Collection
- **Source:** [Kaggle Fashion Product Images](https://www.kaggle.com/datasets/paramaggarwal/fashion-product-images-small) (44,424 products)
- **Augmented with:** occasion, fabric, style, skin_tone, size (rule-based domain knowledge mapping)
- **User ratings:** 9,000 synthetic ratings from 300 simulated users

### 2. Data Cleaning
- Handled missing values (mode/median imputation)
- Removed duplicates
- Standardized text (lowercase, stripped whitespace)
- Added 5 missing columns using domain-knowledge mapping

### 3. Feature Engineering
- **One-Hot Encoding** (scikit-learn) â€” 15 categorical columns â†’ 208 binary features
- **70% / 10% / 20% split** â€” Training / Validation / Test (explicit, not random)
- **StandardScaler** â€” Normalized features for neural network

### 4. Model Training (3 Iterations)

### 5. Evaluation
- Best model (V3): MAE = 0.47 on held-out test set
- Predictions off by less than half a rating point on 1-5 scale
- Learning curves confirm V3 has best generalization (train/val loss stay close)

### 6. Deployment
- **Cosine Similarity Engine** â€” Encodes user preferences into vectors, computes similarity against all 44K products
- **Gender Hard Filter** â€” Wrong gender products are completely removed before scoring
- **Deduplication** â€” Ensures varied recommendations (no 8 identical blue shirts)

### 7. Monitoring
